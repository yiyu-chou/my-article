# Apple 風格 Markdown 撰寫與排版規範指南

本指南旨在規範全站文章與專案的 Markdown 寫作標準，確保所有內容在導入 vanilla 技術渲染引擎後，皆能呈現一致、精準且具呼吸感的 Apple 極簡視覺美學。

---

## 1. 檔案頂部配置 (YAML Front Matter)

每篇 Markdown 文章的**最頂端**必須包含 YAML 前導資料，用以驅動網頁的標題區塊、分類徽章、作者頭像與標籤系統。請務必使用三個短橫線 `---` 包覆，欄位定義如下：

```yaml
---
title_main: "基於智慧邊緣計算的"
title_gradient: "低延遲環境感測整合系統開發"
description: "探討如何整合 ESP32 單晶片、多感測器反饋迴路，並透過極簡自建 API 實現邊緣數據初篩與雲端零延遲同步。"
category: "自主學習 / 工程專案"
tags: [物聯網, 邊緣計算, ESP32, 卡爾曼濾波]
date: 2026-07-18
author: "周奕宇 YiYu Chou"
---

```

### 欄位規範說明：

* **`title_main`**：主標題（純文字，淺色模式下為接近純黑，深色模式下為純白）。
* **`title_gradient`**：漸層強調標題（會接在主標題下方或右側，套用 Apple 科技感藍紫色漸層）。
* **`description`**：文章摘要描述（用於補充說明文章核心，字體略大且呈霧灰色，會取代原本的副標題區塊）。
* **`category`**：文章分類。**必須符合雙層結構，格式為「主分類 / 自訂子項目」**：
* **主分類**僅限四種：`主題研究`、`自主學習`、`活動參與`、`經驗分享`。
* **自訂子項目**由作者依據文章主題自定（例如：`工程專案`、`演算法優化`、`黑客松競賽` 等）。


* **`tags`**：陣列格式，最多建議 **4 個**，系統會自動將其渲染為精緻的灰色膠囊標籤。
* **`date`**：標準格式 `YYYY-MM-DD`，會顯示於作者欄位。
* **`author`**：作者姓名，採「中文姓名 英文姓名」格式。系統會自動解析英文姓名並抓取其大寫首字母組合成圓形高質感文字頭像（例如 `YiYu Chou` 將自動生成頭像字樣 `YC`）。

---

## 2. 標題層級規範 (#, ##, ###)

為了維持畫面的乾淨與層級清晰，全站文章僅啟用三個層級的標題：

### # 一級標題（請勿在內文使用）

* **規範**：內文**嚴禁**使用 `# 標題文字`。
* **原因**：文章主標題已由頂部的 YAML 自動抓取並渲染為最高層級的 H1，內文若重複使用會破壞視覺權重。

### ## 二級標題（核心章節）

* **語法**：`## 1. 工程背景與研究目的`
* **視覺表現**：字體粗大且字距緊湊，左側會自動帶有系統配置的 Apple Blue 垂直點綴條，且會自動被**動態目錄 (TOC) 引擎**抓取。
* **間距**：上方會保留較大留白（`3rem`），下方緊鄰內文。

### ### 三級標題（章節內小節）

* **語法**：`### 3.1 卡爾曼濾波器初始化`
* **視覺表現**：無垂直條點綴，純粹以字體大小與字重與內文做出區隔，**不會**出現在左側目錄中。

## 3. 內文與段落排版

* **段落結尾**：段落與段落之間請**空一行**，渲染引擎會自動套用完美的行高（`1.85`）與下邊距，切勿連續使用多個 `<br>` 來手動推開間距。
* **粗體強調**：使用 `**粗體文字**` 來強調關鍵術語。請適度使用，一頁中粗體比例不宜超過 10%。
* **引言區塊**：使用 `> 引言文字` 來放置核心結論、名言或工程格言。系統會為其加上 Apple Blue 左側邊框與淺灰色微圓角底色，使其具有高度視覺凝聚力。
* **文章起始**：文章一開始請不要使用 `---` 分隔線。

---

## 4. 全自動圖片與圖說規範

本站渲染引擎已內建「標準 Markdown 圖片直出圖說」機制。寫作者**完全不需要**撰寫 HTML 的 `<figure>` 或 `<figcaption>` 標籤。

### 撰寫語法：

```markdown
![圖 1：邊緣計算物聯網硬體拓撲圖](https://example.com/image.jpg)

```

### 排版與渲染機制：

1. **中括號 `[...]` 務必填寫**：裡面的文字會被系統自動抓取，並以 13px 霧灰色字體**置中**呈現在圖片正下方，作為圖片名稱。
2. **間距與特效**：引擎會全自動為圖片加上 `1.5rem` (24px) 的大圓角、極細防吃色邊框與柔和微陰影，並自動將圖片與上下文字的間距緊湊化，確保圖名與照片緊密相依。

## 5. 高質感清單列表 (-)

清單一律使用短橫線 `-` 作為無序清單的開頭（亦可使用標準數字進行有序清單排列）：

```markdown
- **網路頻寬損耗**：傳統直連模式造成伺服器大量無效通訊。
- **伺服器處理延遲**：高頻率取樣使雲端資料庫負荷過重。

```

### 視覺優化點：

* 系統已剔除瀏覽器內建的死板粗大黑點。
* 全自動改用 `5px` 的 Apple Blue 微縮精品圓點，並優化了縮進與行距，文字與列點將完美對齊。

---

## 6. 水平分割線 (---)

當文章要切換重大主題，或是 YAML 結束後進入正文前，請使用三個短橫線獨立一行：

```markdown
---

```

### 視覺優化點：

* 請勿濫用水平線，通常一篇文章出現 **2 至 3 次** 即可。
* 系統將其渲染為兩端羽化消逝（Fade-out）的極細霧面光澤線，在深淺色模式下皆具備極佳的空氣感與呼吸節奏。

---

## 7. 程式碼區塊 (Code Block)

不論是單行程式碼還是多行區塊，皆有專屬優化：

### 單行行內程式碼：

* 語法：使用反單引號，如 ``ESP32-WROOM-32E``。
* 表現：會自動呈現淡淡的背景色與亮藍色字體。

### 多行程式碼區塊：

* 語法：使用三個反單引號並指定語言（例如 ```cpp 或 ```javascript）。
* 表現：系統會將其封裝進一個仿 Apple 終端機外殼的深色容器中，頂部會自動標示 `source_code.語言名稱`，且**右上角會自動生成一個帶有絲滑過渡動畫的「複製代碼」按鈕**。寫作時只需專注於程式碼內容即可。

## 8. 超連結語法與視覺規範

內文中的外部連結、參考文獻或站內跳轉，必須嚴格使用以下標準 Markdown 語法，系統會自動套用 Apple 品牌字色與絲滑的**滑鼠懸浮底線動畫**：

1. **行內連結（最常用）**：
```markdown
詳細技術規格請參閱 [ESP32 官方技術手冊](https://espressif.com)。

```

2. **參考式連結（適合長網址或重複使用）**：
```markdown
關於核心功能請參考 [引腳定義][esp32-spec]，此元件的 [低功耗模式][esp32-spec] 同樣符合專案需求。

[esp32-spec]: [https://example.com/very-long-url-to-spec](https://example.com/very-long-url-to-spec)

```

* **注意**：文字內容與中括號之間請勿留白，網址必須包含 `https://`（若為站內相對路徑則免，如 `[關於我](index.html#about)`）。
