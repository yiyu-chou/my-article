## 專案背景

隨著物聯網（Internet of Things, IoT）、人工智慧（Artificial Intelligence, AI）以及雲端運算技術快速發展，智慧校園逐漸成為現代教育的重要方向。傳統校園多依賴人工管理，不僅效率有限，也難以即時掌握各項設備與環境資訊。

本專案希望以「**從零構建智慧校園**」為目標，透過感測器、自動化控制、資料分析及網頁平台，打造一套整合性的智慧校園管理系統，讓校園更加安全、節能且便利。

> 智慧校園的核心，不只是科技的堆疊，而是透過數據讓校園運作更加高效與人性化。

---

## 專案目標

本系統主要目標如下：

- 建立即時環境監測平台
- 整合校園各項設備資訊
- 建置雲端資料管理系統
- 提供管理者即時決策依據
- 提升學生與教師的使用體驗
- 降低能源浪費與維護成本

透過上述功能，希望讓校園管理由傳統模式轉變為智慧化、自動化管理。

---

## 系統架構

整體系統可分為四個層級：

### 感測層（Sensor Layer）

負責收集校園環境資訊，例如：

- 溫度
- 濕度
- 空氣品質（PM2.5）
- 二氧化碳濃度
- 光照強度
- 人體紅外線感測
- 用電資訊

各項資料由 ESP32 或其他微控制器負責收集。

---

### 網路傳輸層（Network Layer）

感測器資料透過 Wi-Fi 傳送至雲端。

常見通訊方式包括：

- HTTP API
- MQTT
- WebSocket

資料可即時同步至資料庫，供後續分析使用。

---

### 雲端平台層（Cloud Layer）

伺服器主要負責：

- 接收感測資料
- 儲存資料庫
- 使用者登入驗證
- API 管理
- 資料分析
- 異常通知

可採用：

- Firebase
- Supabase
- MySQL
- PostgreSQL
- Node.js API

建立完整的後端服務。

---

### 應用層（Application Layer）

提供使用者操作介面，例如：

- 校園即時監控儀表板
- 手機網頁
- 管理後台
- 數據圖表分析
- 設備控制介面

管理者可透過瀏覽器查看所有校園設備狀態。

---

## 核心功能

### 即時環境監測

系統持續收集教室環境資訊，包括：

- 溫度
- 濕度
- PM2.5
- CO₂
- 光照

若數值超出設定範圍，即可立即發送通知。

例如：

- 教室過熱
- 空氣品質不佳
- 二氧化碳過高

提醒教師開窗或啟動空調。

---

### 智慧照明

利用人體感測器控制照明：

- 有人自動開燈
- 無人自動關燈
- 夜間降低亮度
- 節省能源

有效降低校園用電成本。

---

### 智慧能源管理

系統可即時監測：

- 教室耗電量
- 每日用電
- 每月用電
- 尖峰用電

並以圖表方式呈現。

管理者能快速找出高耗能設備。

---

### AI 異常預測

透過歷史資料建立模型，可預測：

- 設備故障
- 異常耗電
- 教室空氣品質惡化

提前通知管理人員進行維護。

---

### 校園公告整合

智慧校園平台可整合：

- 校園公告
- 課程資訊
- 活動通知
- 緊急警報

學生登入即可查看最新資訊。
