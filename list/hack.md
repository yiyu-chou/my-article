## 競賽簡介

**黑客松（Hackathon）** 是一種結合創新、技術與團隊合作的競賽活動，參賽者需要在限定時間內，針對指定主題提出解決方案，並完成可展示的作品原型（Prototype）。

近年來，高中黑客松逐漸普及，競賽內容不再侷限於程式設計，也包含 **人工智慧（AI）**、**物聯網（IoT）**、**永續發展（SDGs）**、**智慧醫療**、**教育科技** 等跨領域主題，鼓勵學生運用科技解決生活中的問題。

> 黑客松不只是程式競賽，更是一場創意、技術與團隊合作能力的挑戰。

---

## 競賽流程

高中黑客松通常會依照以下流程進行：

### 1. 主題公布

競賽開始時，主辦單位會公布年度主題，例如：

- 智慧校園
- 智慧交通
- AI 應用
- 永續環境
- 數位健康

團隊必須在短時間內發想解決方案。

### 2. 團隊討論

隊員共同討論：

- 問題分析
- 使用者需求
- 解決方法
- 系統架構

此階段通常會利用便利貼、心智圖或白板整理想法。

### 3. 系統開發

確定方向後開始分工，例如：

- 前端介面設計
- 後端 API
- 資料庫建置
- AI 模型訓練
- 簡報製作

大家同步合作完成作品。

### 4. 成果發表

每組需向評審展示作品，包括：

- 問題背景
- 解決方案
- 系統展示
- 技術特色
- 未來發展

最後由評審依據創新性、完整度、技術能力及簡報表現進行評分。

---

## 團隊合作的重要性

黑客松最大的特色，就是**團隊合作**。

一個成功的作品通常需要不同專長的人共同完成，例如：

- **程式開發者**：負責系統功能實作。
- **UI／UX 設計**：設計操作介面。
- **企劃人員**：分析需求與規劃功能。
- **簡報負責人**：整理成果並向評審介紹。

每位成員都有自己的任務，也必須持續溝通，才能讓專案順利完成。

---

## 技術能力的提升

參加黑客松能快速提升許多實作能力，例如：

- Python 程式設計
- JavaScript 網頁開發
- Git 版本控制
- RESTful API
- Firebase 雲端資料庫
- AI 模型應用
- Figma UI 設計

除了課堂知識之外，更重要的是學會如何把不同技術整合成完整的系統。

---

## 面臨的挑戰

黑客松通常只有 **24～48 小時** 的開發時間，因此會遇到許多挑戰。

### 時間壓力

有限時間內必須完成：

- 發想
- 設計
- 開發
- 測試
- 簡報

因此良好的時間管理十分重要。

### 技術問題

常見問題包括：

- API 無法連線
- 程式錯誤（Bug）
- 資料格式不一致
- AI 模型辨識率不足

團隊需要快速查詢資料並共同解決問題。

### 團隊溝通

不同成員可能有不同想法，因此需要：

- 尊重彼此意見
- 清楚分工
- 定期同步進度

才能避免重複工作或延誤開發。

---

## 競賽收穫

透過黑客松，不僅能提升技術能力，也能培養許多軟實力。

- **問題解決能力**：快速分析並提出可行方案。
- **團隊合作能力**：學習有效分工與協調。
- **簡報能力**：清楚介紹自己的作品。
- **創新思考能力**：從不同角度思考問題。
- **抗壓能力**：在有限時間內完成高品質成果。

這些能力對未來升學、專題研究及職涯發展都具有重要價值。

---

## 個人心得

高中黑客松是一個將課堂知識轉化為實際作品的重要舞台。在競賽過程中，不僅能學習新的技術，更能體驗團隊合作與專案開發的完整流程。雖然過程中會遇到許多困難與挑戰，但每一次解決問題的經驗，都能成為成長的重要養分。

未來希望持續參與更多不同主題的黑客松活動，累積實戰經驗，提升跨領域整合能力，並透過科技創新，為生活中的問題提出更多具有價值的解決方案。

---

## 結語

黑客松競賽不只是技術能力的比拼，更是一場創意、合作與實踐能力的綜合考驗。對高中生而言，它提供了一個將想法化為成果的平台，也讓學生在有限時間內學習如何快速解決問題、有效溝通及持續創新。透過一次次的參與，不僅能累積作品與經驗，更能為未來的大學學習及職涯發展奠定良好的基礎。
